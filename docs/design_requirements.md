## Design Requirements

| **ID** | **Categorie** | **Design requirement** | **Source** |
|--------|---------------|------------------------|------------|
| **F1** | **Auditieve Interactie & Muzikale Functionaliteit** | Het product stuurt hoofdzakelijk auditieve feedback naar de gebruiker. | Interview + Literatuur |
| **F2** |  | Het product biedt ondersteuning van relatief en absoluut gehoor. | interview + Literatuur |
| **F3** |  | Het product maakt gebruik van talking-scores (audio partituur). | User-Test 1/2 |
| **F4** |  | Het product heeft functies om de audio partituur te kunnen besturen. | User-Test 1/2 |
| **F5** |  | Het product kan een audio partituur pauzeren/hervatten. | User-Test 1/2 |
| **F6** |  | Het product kan een audio partituur door- en terugspoelen. | User-Test 1/2 |
| **F7** |  | Het product geeft indicaties tijdens het door- en terugspoelen. | User-Test 1/2 |
| **F8** |  | Het product kan schakelen tussen een audio partituur en een audiotrack. | User-Test 1/2 |
| **F9** |  | Het product biedt een "loop" functie voor het intensief leren van moeilijkere stukken muziek. | User-Test 1 |
|  |  |  |  |
| **L1** | **Leerondersteuning & Oefenproces** | Het product biedt een stapsgewijs leerpad. | Interview |
| **L2** |  | Het product beperkt onderbrekingen tijdens het oefenen. | Literatuur |
| **L3** |  | De app biedt een auditieve tutorial. | User-Test 1 |
| **L4** |  | De app kan de auditieve tutorial aan en uit zetten. | User-Test 1 |
|  |  |  |  |
| **U1** | **Gebruiksvriendelijkheid & Cognitieve Belasting** | Het product is een lage cognitieve belasting tijdens het gebruik. | Literatuur |
| **U2** |  | Het product biedt intuïtieve inputs voor iedere functie. | User-Test 2 |
| **U3** |  | Het product is bruikbaar zonder voorafgaande uitleg of training. | User-Test 1/2 |
| **U4** |  | Het product beperkt het aantal primaire bedieningsinputs om complexiteit en foutgebruik te verminderen. | User-Test 2 |
| **U5** |  | De app biedt een intuïtieve bediening. | User-Test 1/2 |
| **U6** |  | De app biedt een layout waar de belangrijkste inputs niet van plaats veranderen. | User-Test 2 |
| **U7** |  | De app speelt in op gewoontes van de gebruiker en onnodige functies vermijden. | User-Test 2 |
| **U8** |  | De app geeft aan hoeveel knoppen er per interface zijn om zo het aantal mogelijke inputs te verduidelijken. | User-Test 2 |
|  |  |  |  |
| **A1** | **Toegankelijkheid & Inclusief Ontwerp** | Het product is bestuurbaar met minimale tactiele leesvaardigheid. | Interview |
| **A2** |  | Het product kan bestuurd worden zonder handen. | User-Test 1 |
| **A3** |  | De app heeft gelabelde knoppen om makkelijke oriëntatie met voice over te verkrijgen. | User-Test 1 |
|  |  |  |  |
| **T1** | **Fysieke & Technische Randvoorwaarden** | Het product biedt een stabiele rust positie aan de gebruiker (ergonomisch). | User-Test 1 |
| **T2** |  | Het product biedt signifiers om de opstelling correct te plaatsen. | User-Test 2 |
| **T3** |  | Het product is compatibel met bestaande muziekformaten. | Literatuur |
